# Assets

Collection of reusable assets: prompts, templates, visuals and documents.

## Purpose

This repository centralizes reusable materials to support marketing strategy, content creation and AI-driven workflows.

It is designed as a structured knowledge base to:
- save time on recurring tasks
- ensure consistency in thinking and execution
- continuously improve quality through iteration

---

## Structure

The repository is organized by asset type:

### /prompts
Reusable prompts for AI tools.

Examples:
- strategy prompts
- copywriting prompts
- analysis frameworks
- SEO / content generation

---

### /templates
Reusable templates for marketing and communication.

Examples:
- presentation structures
- strategic recommendations
- briefs
- landing pages

---

### /frameworks
Structured methods and thinking models.

Examples:
- consumer insight frameworks
- brand platform structures
- messaging hierarchies
- funnel models

---

### /copy-examples
Curated writing examples and patterns.

Examples:
- headlines
- hooks
- scripts
- long-form content

---

### /profiles
Target, audience or brand profiles.

Examples:
- personas
- brand positioning
- tone of voice definitions

---

### /assets
Visual and supporting materials.

Examples:
- images
- diagrams
- PDF documents

---

## How to use

Each file is designed to be:
- directly reusable
- easy to adapt
- quickly understandable

Recommended workflow:
1. Identify the need (strategy, copy, analysis…)
2. Browse the relevant folder
3. Copy and adapt the asset
4. Improve and version if needed

---

## Principles

- **Clarity over complexity**  
  Each asset should be immediately understandable.

- **Reusability first**  
  Content must be actionable and adaptable.

- **One idea per file**  
  Avoid mixing multiple concepts.

- **Continuous improvement**  
  Assets are meant to evolve over time.

---

## Naming conventions

Files should be named clearly and explicitly:

Examples:
- `consumer-insight-framework.md`
- `luxury-tone-of-voice.md`
- `landing-page-structure.md`

Avoid:
- vague names (`test`, `draft`, `final_v2`)

---

## Notes

- Sensitive or client-specific content should be anonymized.
- This repository is a living system, not an archive.
